echolotl.lol backend API thing

Hello! This is the backend API for echolotl.lol. You can use these endpoints however you like, but please don't abuse them. There are rate limits.

This is a mega WIP as I learn Elysia and backend stuff. You can view the source code here.

Last updated 3/10/2026.

Documentation

  • GET /spotify
    Gets echolotl's current playing song on Spotify.

    Gets echolotl's current playing song on Spotify.

    Rate Limit

    15 requests over 1 minute

    Returns

    200 OK

    Returns the currently playing song on Spotify, if any.

    Type
    Example
    204 No Content Returns if echolotl is not currently playing anything on Spotify.
  • GET /status
    Gets echolotl's latest status.

    Gets echolotl's latest status.

    Rate Limit

    15 requests over 2 minutes

    Returns

    200 OK

    Returns the latest status object.

    Example
    204 No Content Returns when no status has been set yet. (Which probably means that the database got deleted. Whoops!)
  • GET /status/history
    Gets recent status history.

    Gets recent statuses, newest first.

    Query Parameters

    limit (optional number): Number of statuses to return. Defaults to 10.

    Rate Limit

    15 requests over 2 minutes

    Returns

    200 OK

    Returns a list of statuses and the count returned.

    Example
    { "count": 2, "statuses": [ { "text": "sleepy", "emoji": "😴", "createdAt": 1741553075123 }, { "text": "working on the backend api", "emoji": null, "createdAt": 1741553070000 } ] }
    204 No Content Returns when no statuses are available.
  • POST /status lock
    Sets a new status.

    Creates a new status entry.

    Headers

    Authorization: required passkey token. (Defined in server config)

    Body

    {
  "text": "string (1-100 chars)",
  "emoji": "emoji (optional, nullable)"
}

    Rate Limit

    15 requests over 2 minutes

    Returns

    200 OK Returns a confirmation text string with the new status.
    401 Unauthorized Missing Authorization header.
    403 Forbidden Provided passkey is invalid.
