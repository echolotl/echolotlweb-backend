echolotl.lol backend API thing docs

Hello! This is the backend API for echolotl.lol. You can use these endpoints however you like, but please don't abuse them. There are rate limits.

This is a mega WIP as I learn Elysia and backend stuff. You can view the source code here.

Last updated 7/8/2026.

    Gets echolotl's current playing song on Spotify.

    Gets echolotl's current playing song on Spotify.

    Rate Limit

    15 requests over 1 minute

    Returns

    200 OK

    Returns the currently playing song on Spotify, if any.

    Type
    Example

  • GET /status
    Gets echolotl's latest status.

    Gets echolotl's latest status.

    Rate Limit

    15 requests over 2 minutes

    Returns

    Returns the latest status object.

    Example

    Gets recent status history.

    Gets recent statuses, newest first.

    Query Parameters

    limit (optional number): Number of statuses to return. Defaults to 10.

    cursor (optional number): The ID of the last status from the previous page, used for pagination.

    Rate Limit

    Returns

  • POST /status lock
    Sets a new status.

    Creates a new status entry.

    Headers

    Requires an Authorization: Bearer <PASSKEY> header, where PASSKEY matches the server's configured PASSKEY environment variable.

    Body

    {
  "text": "string (1-100 chars)",
  "emoji": "emoji (optional, nullable)"
}

    Rate Limit

    15 requests over 2 minutes

    Returns

    401 Unauthorized Missing or invalid Authorization bearer token.

Discord OAuth

Lets you login with Discord for echolotl.lol. This is what will power any authenticated requests.

  • GET /auth/discord/
    Starts the Discord login flow.

    Redirects the browser to Discord's OAuth consent screen. Sets a short-lived oauth_state cookie for validation.

    Rate Limit

    20 requests over 1 minute, shared across all Discord auth routes

    Returns

    302 Found Redirects to Discord's OAuth authorize page.
  • GET /auth/discord/authenticate
    OAuth callback used by Discord.

    This is the redirect_uri configured on the Discord application and it's not meant to be called directly. Discord redirects here with a code and state after the user approves the login.

    This does more additional checks to ensure the Discord account is at least a month old (to mitigate spam), and then saves the Discord user info to the database and sets a session cookie for future authenticated requests.

    Query Parameters

    code (string): Authorization code provided by Discord.
    state (string): Must match the oauth_state cookie set in /auth/discord/.

    Rate Limit

    20 requests over 1 minute, shared across all Discord auth routes

    Returns

    302 Found Login succeeded. Redirects to the frontend with a session cookie set.
    400 Bad Request Missing or invalid query parameters.
    403 Forbidden The Discord account doesn't meet the minimum account age requirement.
    502 Bad Gateway Failed to reach Discord's token exchange or user info endpoints.
  • GET /auth/discord/logout
    Logs out the current session.

    Destroys the session tied to the session cookie (if any) and clears the cookie.

    Rate Limit

    20 requests over 1 minute, shared across all Discord auth routes

    Returns

    200 OK

    Returns a confirmation text string.
  • GET /auth/discord/me
    Gets the currently logged in user.

    Reads the session cookie and returns the associated user as a full user object. This always returns the full user object, even if the user has enabled anonymous mode.

    Rate Limit

    20 requests over 1 minute, shared across all Discord auth routes

    Returns

    200 OK

    Anonymous mode disabled:

    Anonymous mode enabled:
    204 No Content Returns if not logged in.
  • PATCH /auth/discord/me lock
    Toggles anonymous mode for the current user.

    Requires an active session cookie. Can be used to set the anonymous mode for the current user, and returns a full user object with the updates. This is a non-anonymous object for the same reason as the GET /auth/discord/me endpoint.

    Body

    Rate Limit

    20 requests over 1 minute, shared across all Discord auth routes

    Returns

    200 OK

    Returns the updated user.
    401 Unauthorized Returns if not logged in.
    404 Not Found Returns if the user no longer exists.
  • DELETE /auth/discord/me lock
    Deletes the current user and their data.

    Requires an active session cookie. This will delete the current user and all their data from the database.

    Rate Limit

    20 requests over 1 minute, shared across all Discord auth routes

    Returns

    200 OK

    Returns a confirmation text string.
    401 Unauthorized Returns if not logged in.
    404 Not Found Returns if the user doesn't exist.
  • GET /auth/discord/user/:userId
    Looks up a user by their (non-Discord) user ID.

    Returns a user object for the given userId. If the user has enabled anonymous mode, this will return a redacted user object with a generic username and display name, and no avatar hash.

    Path Parameters

    userId (string): The user's userId, as returned by /auth/discord/me or itself.

    Rate Limit

    20 requests over 1 minute, shared across all Discord auth routes

    Returns

    200 OK

    Anonymous mode disabled:

    Anonymous mode enabled:
    404 Not Found Returns if no user has that userId.

Comments

Lets people leave comments (and replies) on art pieces (or, theoretically, anything in the future), identified by a slug. Posting, editing, and deleting comments requires an active session cookie from Discord OAuth. The slugs provided aren't checked for existence in actuality.

  • GET /comments/user/:userId
    Gets a user's recent comments.

    Returns the user's public profile along with their most recent comments, newest first. Comments don't include author info here since it's the same user for all of them.

    Path Parameters

    userId (string): The user's standalone userId.

    Query Parameters

    limit (optional number): Number of comments to return. Defaults to 10.

    Rate Limit

    100 requests over 1 minute, shared across all comment routes

    Returns

    200 OK

    Returns the user and their comments.

    Example
    404 Not Found Returns if no user has that userId.
  • GET /comments/s/:slug
    Gets top-level comments for a slug.

    Returns the top-level (non-reply) comments for the given slug, oldest first. Each comment includes a replyCount so you can fetch replies separately from GET /comments/:id/replies.

    Path Parameters

    slug (string): The slug of the art piece being commented on.

    Query Parameters

    limit (optional number): Number of top-level comments to return. Defaults to 10, capped at 50.

    cursor (optional number): The id of the last comment from the previous page. Pass the response's nextCursor to fetch the next page.

    Rate Limit

    100 requests over 1 minute, shared across all comment routes

    Returns

    200 OK

    Returns the slug, a page of top-level comments, and a nextCursor (or null if there are no more pages).

    Example
  • POST /comments/s/:slug lock
    Posts a new top-level comment.

    Requires an active session cookie. Creates a new top-level comment on the given slug.

    Path Parameters

    slug (string): The identifier for the art piece being commented on.

    Body

    Plain text body (Content-Type: text/plain), 1-5000 characters.

    Rate Limit

    100 requests over 1 minute, shared across all comment routes

    Returns

    201 Created

    Returns the newly created comment.

    Example
    401 Unauthorized Returns if not logged in.
  • GET /comments/:id
    Gets a single comment.

    Returns a single comment by id, along with its replyCount. Fetch replies separately from GET /comments/:id/replies.

    Path Parameters

    id (number): The comment's numeric ID.

    Rate Limit

    100 requests over 1 minute, shared across all comment routes

    Returns

    200 OK

    Returns the comment.

    Example
    400 Bad Request Returns if id isn't an integer.
    404 Not Found Returns if no comment with that id exists.
  • GET /comments/:id/replies
    Gets replies to a comment.

    Returns the direct replies to the comment id, oldest first. Each reply includes its own replyCount; fetch nested replies by calling this route again with the reply's own id.

    Path Parameters

    id (number): The parent comment's ID.

    Query Parameters

    limit (optional number): Number of replies to return. Defaults to 10, capped at 50.

    cursor (optional number): The id of the last reply from the previous page. Pass the response's nextCursor to fetch the next page.

    Rate Limit

    100 requests over 1 minute, shared across all comment routes

    Returns

    200 OK

    Returns the parent comment's id, a page of replies, and a nextCursor (or null if there are no more pages).

    Example
    400 Bad Request Returns if id isn't a valid integer.
    404 Not Found Returns if no comment with that id exists.
  • POST /comments/:id lock
    Posts a reply to a comment.

    Requires an active session cookie. Creates a reply to the comment id, inheriting its slug.

    Path Parameters

    id (number): The parent comment's numeric ID.

    Body

    Plain text body (Content-Type: text/plain), 1-5000 characters.

    Rate Limit

    100 requests over 1 minute, shared across all comment routes

    Returns

    201 Created

    Returns the newly created reply.

    Example
    400 Bad Request Returns if id isn't an integer.
    401 Unauthorized Returns if not logged in.
    404 Not Found Returns if no parent comment with that id exists.
  • PATCH /comments/:id lock
    Edits one of your own comments.

    Requires an active session cookie. You can only edit your own comments, and only if they haven't been deleted.

    Path Parameters

    id (number): The comment's numeric ID.

    Body

    Plain text body (Content-Type: text/plain), 1-5000 characters.

    Rate Limit

    100 requests over 1 minute, shared across all comment routes

    Returns

    200 OK

    Returns the updated comment.

    Example
    400 Bad Request Returns if id isn't a valid integer.
    401 Unauthorized Returns if not logged in.
    403 Forbidden Returns if the comment isn't yours.
    404 Not Found Returns if no comment with that id exists.
    410 Gone Returns if the comment has already been deleted.
  • DELETE /comments/:id lock
    Deletes one of your own comments.

    Requires an active session cookie. You can only delete your own comments. The comment's body is replaced with [deleted] rather than being removed entirely, so replies stay intact.

    Path Parameters

    id (number): The comment's numeric ID.

    Rate Limit

    100 requests over 1 minute, shared across all comment routes

    Returns

    204 No Content Comment deleted successfully (or was already deleted).
    400 Bad Request Returns if id isn't a valid integer.
    401 Unauthorized Returns if not logged in.
    403 Forbidden Returns if the comment isn't yours.
    404 Not Found Returns if no comment with that id exists.
